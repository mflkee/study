# Домашнее задание к теме «Структуры, enum, методы и traits»

В исходном JavaScript-репозитории в этом месте была тема про классы. Для Rust
это слишком прямолинейный перенос, поэтому задание перестроено под идиоматичный
подход языка.

Здесь нужно мыслить не наследованием, а:

- `struct` для данных;
- `enum` для состояний и видов;
- `impl` для поведения;
- `trait` для общего интерфейса.

## Задача. Каталог учебной библиотеки

Нужно описать библиотеку учебных материалов.

### Что нужно сделать

1. Создайте `enum ItemType` для вида материала.
2. Создайте `enum Condition` для состояния экземпляра.
3. Создайте `struct LibraryItem`.
4. Реализуйте конструкторы:
   - `LibraryItem::book`;
   - `LibraryItem::magazine`;
   - `LibraryItem::lecture_notes`.
5. Создайте `trait Describable` с методами:
   - `describe`;
   - `can_be_lent`.
6. Реализуйте `struct Library` и методы:
   - `new`;
   - `add_item`;
   - `find_by_title`;
   - `lend`;
   - `items_by_type`.

## Важное ограничение

Не пытайтесь имитировать иерархию классов JavaScript один в один. Здесь ценность
задания в том, чтобы увидеть, как Rust-структуры и `enum` заменяют наследование.

## Результат при правильном решении

Все тесты из `tests/library.rs` проходят успешно.

## Требования к выполнению

1. Код проходит `cargo test`.
2. Код форматируется через `cargo fmt`.
3. Код проходит `cargo clippy --all-targets --all-features -- -D warnings`.

## Повышенный уровень сложности

1. Добавьте отдельный метод для подсчёта материалов, которые нельзя выдавать.
2. Реализуйте сортировку найденных материалов по году.
