# Итоговое задание по курсу Rust

## Проект: CLI-планировщик учебной нагрузки

Нужно спроектировать и реализовать консольное приложение, которым студент может пользоваться каждый день для управления учебными задачами.

Это итоговая работа, поэтому здесь уже не должно быть механического копирования решений из практики. Вам нужно собрать самостоятельный проект, в котором осмысленно применяются возможности Rust.

## Обязательные возможности

### Модель предметной области

Реализуйте:

- `Task` или `StudyTask`;
- `Priority`;
- `Status`;
- `AppError` или аналогичный тип ошибок.

### Команды

Поддержите минимум:

1. `add`
2. `list`
3. `done`
4. `remove`
5. `stats`
6. `help`

### Хранение данных

Данные должны сохраняться между запусками программы.

Допустимые варианты:

- текстовый формат;
- CSV;
- JSON.

### Организация проекта

Минимум:

- отдельный модуль для доменной модели;
- отдельный модуль для хранения;
- отдельный модуль для CLI-логики или разбора команд;
- тесты.

### Обработка ошибок

Проект не должен падать на штатных ошибках ввода.

Нужно корректно обрабатывать:

- неизвестную команду;
- пустой заголовок;
- некорректный индекс;
- ошибку чтения или записи файла;
- повреждённую строку данных.

## Обязательные технические требования

1. `cargo fmt --check` проходит.
2. `cargo clippy --all-targets --all-features -- -D warnings` проходит.
3. `cargo test` проходит.
4. В проекте есть unit tests.
5. В проекте есть хотя бы один integration test.
6. В проекте есть хотя бы один doc test.

## Что будет считаться сильным решением

- понятная доменная модель;
- отсутствие лишних `clone`;
- разумное использование `&str`, `String`, `Vec`, `HashMap`;
- отсутствие бессистемного `unwrap`;
- хороший `match` по командам и ошибкам;
- ясное разделение на модули;
- чистый и предсказуемый формат хранения.

## Дополнительные возможности

Это не обязательно, но усиливает проект:

1. Фильтрация по приоритету и статусу.
2. Поиск по подстроке.
3. Сортировка по приоритету или сроку.
4. Поддержка тегов.
5. Переход на `clap` для CLI.
6. Переход на `serde` для JSON.
7. Разделение на workspace из `core` и `cli`.

## Как защищать работу

Студент должен уметь объяснить:

- почему структура проекта выбрана именно так;
- где и зачем используется `Result`;
- где заимствование лучше владения;
- почему модель сделана через `enum`, а не через магические строки;
- какие тесты покрывают ключевые сценарии.
