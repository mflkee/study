# Домашнее задание к теме «Парсинг логов и отчёты»

## Задача. Анализ журнала событий учебной платформы

Нужно реализовать небольшой модуль, который разбирает лог-файл и строит по нему отчёты.

Формат строки:

```text
ERROR|2026-03-10T10:15|auth|invalid token
```

## Что нужно сделать

1. Реализовать `enum LogLevel`.
2. Реализовать `struct LogEntry`.
3. Реализовать `enum LogParseError`.
4. Реализовать функцию `parse_entry`.
5. Реализовать функции отчёта:
   - `count_by_level`;
   - `modules_with_errors`;
   - `latest_message_for`.

## Важно

- Повреждённые строки должны приводить к `Err`, а не к `panic!`.
- Отчёты должны работать по уже распарсенным данным.
- Не смешивайте парсинг и отчётную логику в одну функцию.

## Результат при правильном решении

Все тесты из `tests/log_reports.rs` проходят успешно.

## Повышенный уровень сложности

1. Добавьте функцию, которая строит частоту ошибок по модулям через `HashMap`.
2. Реализуйте функцию чтения и разбора целого файла.
