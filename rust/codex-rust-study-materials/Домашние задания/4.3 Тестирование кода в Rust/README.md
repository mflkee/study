# Домашнее задание к теме «Тестирование кода в Rust»

## Задача

Возьмите проект из предыдущих заданий и покройте ключевую логику тестами трёх типов:

- unit tests;
- integration tests;
- doc tests.

## Что нужно сделать

1. Написать unit tests для основной доменной логики.
2. Написать integration test для публичного API.
3. Добавить хотя бы один doc test в публичную функцию.
4. Убедиться, что `cargo test` проходит полностью.

## Что проверять

- парсинг команд;
- обработку ошибок;
- операции добавления, удаления, завершения задач;
- форматирование или подготовку отчёта.
