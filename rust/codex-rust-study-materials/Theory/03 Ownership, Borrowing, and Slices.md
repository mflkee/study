# 03 Ownership, borrowing и slices

## Почему это центральная тема Rust

Ownership и borrowing не являются “одной из тем”. Это фундамент всей модели
памяти языка. Если студент не понимает владение, он будет упираться почти в
каждый нетривиальный кусок кода.

## Три базовых правила ownership

1. У каждого значения есть владелец.
2. Одновременно владелец только один.
3. Когда владелец выходит из области видимости, значение освобождается.

Пример перемещения:

```rust
let first = String::from("Rust");
let second = first;
```

После этого `first` больше использовать нельзя: владение переехало в `second`.

## Borrowing

Если функция не должна забирать владение, передавайте ссылку:

```rust
fn len_of(text: &str) -> usize {
    text.len()
}
```

Изменяемое заимствование:

```rust
fn add_bang(text: &mut String) {
    text.push('!');
}
```

Ключевой запрет: одновременно нельзя иметь много конфликтующих ссылок на одно и
то же значение.

## Почему borrow checker полезен

Он не “мешает писать код”, а не даёт оставить за собой:

- use-after-free;
- висячие ссылки;
- часть гонок данных ещё до запуска программы.

## Slices

Срезы позволяют заимствовать часть данных без копирования:

```rust
let text = String::from("cargo test");
let first = &text[..5];
```

То же для векторов:

```rust
let values = vec![1, 2, 3, 4];
let head = &values[..2];
```

## Частые ошибки

- Принимать `String`, когда достаточно `&str`.
- Делать `clone()` как универсальный способ “успокоить компилятор”.
- Держать mutable borrow слишком долго.
- Не понимать, что срезы зависят от времени жизни исходных данных.

## Практический ориентир

Перед каждой новой функцией задавайте вопрос:

- ей действительно нужно владеть данными?
- ей нужна только ссылка?
- ей нужен mutable borrow?
