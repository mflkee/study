# 01 Tools, rustup, and Cargo

## Зачем это нужно

В Rust очень многое завязано не только на язык, но и на дисциплину работы с
инструментами. Если студент с самого начала понимает `cargo`, тесты,
форматирование и структуру проекта, дальше обучение идёт заметно ровнее.

## Что обязательно понимать

- `rustc` компилирует код;
- `cargo` создаёт, собирает, запускает и тестирует проекты;
- `rustfmt` форматирует код;
- `clippy` ловит многие плохие инженерные решения;
- проект Rust почти всегда начинается с `cargo new`.

## Базовый цикл

```bash
cargo new my-app
cd my-app
cargo check
cargo run
cargo test
cargo fmt
cargo clippy --all-targets --all-features -- -D warnings
```

`cargo check` особенно важен: он быстрее полной сборки и позволяет часто
проверять код в процессе написания.

## Как устроен проект

Минимальный бинарный проект:

```text
my-app/
  Cargo.toml
  src/
    main.rs
```

Если вам нужна библиотека:

```text
my-lib/
  Cargo.toml
  src/
    lib.rs
```

В реальной разработке библиотечная логика почти всегда удобнее для тестирования,
чем код, написанный прямо в `main.rs`.

## Что такое `Cargo.toml`

Это манифест проекта. В нём лежат:

- имя пакета;
- версия;
- edition;
- зависимости;
- иногда настройки features, profiles и workspace.

Edition `2024` определяет набор правил языка и рекомендуемый современный стиль
проекта.

## Типичные ошибки новичков

- Писать всё в `main.rs`, а потом мучиться с тестами.
- Проверять код только через `cargo run`, игнорируя `cargo check`.
- Не запускать `clippy`, а потом закреплять плохие привычки.
- Считать, что структура проекта не важна “пока код маленький”.
