# 08 CLI, Files, and Data Parsing

## Почему эта тема важна

Для учебного Rust естественным полем практики являются консольные приложения.
Они быстро дают пользу и заставляют по-настоящему применять строки, парсинг,
ошибки, коллекции и организацию кода.

## Аргументы командной строки

Простейший путь:

```rust
let args: Vec<String> = std::env::args().collect();
```

Но уже на втором-третьем проекте лучше выносить разбор в отдельную функцию или
модуль.

## Чтение и запись файлов

```rust
use std::fs;

let content = fs::read_to_string(path)?;
fs::write(path, output)?;
```

На старте этого достаточно. Для больших файлов позже можно переходить к
`BufRead`, потоковой обработке и более сложным форматам.

## Разбор строк

Очень частый учебный паттерн:

1. разделить строку по разделителю;
2. проверить количество частей;
3. привести поля к нужным типам;
4. вернуть `Result`.

Пример:

```rust
let parts: Vec<&str> = line.split('|').collect();
```

Дальше почти всегда имеет смысл сразу переходить от “сырых строк” к
typed-модели.

## Форматы хранения

Для учебных задач подойдут:

- простой текстовый формат;
- CSV;
- JSON.

На старте даже примитивный формат вроде `0|Купить книгу` уже полезен, если он
стабилен и хорошо тестируется.

## Частые ошибки

- Смешивать чтение файла, парсинг и бизнес-логику в одной функции.
- Не различать ошибки ввода и ошибки хранения.
- Парсить “на глаз” без тестов на плохой ввод.
- Делать CLI-логику слишком толстой.
