# 11 Параллелизм: threads, channels и shared state

## Почему эту тему стоит включать в базу

Даже если студент не станет сразу писать сложный concurrent-код, ему полезно понять, что Rust даёт безопасные примитивы параллелизма и почему это считается одной из сильных сторон языка.

## Потоки

Простейший запуск:

```rust
use std::thread;

let handle = thread::spawn(|| 2 + 2);
let result = handle.join().unwrap();
```

Здесь уже видно, что поток возвращает значение через `JoinHandle`.

## Каналы

Когда нужно передавать данные между потоками:

```rust
use std::sync::mpsc;

let (tx, rx) = mpsc::channel();
tx.send(42).unwrap();
let value = rx.recv().unwrap();
```

Каналы полезны, когда хочется отделить производителей данных от потребителей.

## Shared state

Иногда несколько потоков должны менять общее состояние. Для этого обычно используют:

- `Arc<T>` для разделяемого владения;
- `Mutex<T>` для безопасного доступа к изменяемым данным.

```rust
use std::sync::{Arc, Mutex};
```

Важно: Rust не убирает необходимость думать про логику многопоточности, но делает многие ошибки заметно сложнее.

## Где это реально пригодится в учебных задачах

- параллельный подсчёт сумм;
- обработка больших наборов строк;
- распределение задач по рабочим потокам;
- построение простого producer-consumer pipeline.

## Частые ошибки

- Пытаться распараллелить всё подряд без профита.
- Брать `Arc<Mutex<_>>` там, где хватило бы простого разбиения данных по потокам.
- Игнорировать цену синхронизации.
- Писать многопоточный код до того, как хорошо понятна однопоточная версия.
